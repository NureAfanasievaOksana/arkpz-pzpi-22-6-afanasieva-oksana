5.1	 Мета роботи
Розгорнути програмну систему та продемонструвати її роботу.
5.2	 Хід роботи
5.2.1 Розгорнути реалізовану програмну систему
Для налаштування бази даних був створений акаунт Azure та ресурсна група під назвою SortGarbage, де було створено новий сервер бази даних Azure Database for PostgreSQL.
Для розгортання API в Azure був використаний репозиторій проєкту, підготовлений для публікації. У файлі appsettings.json були налаштовані параметри підключення до бази даних, включаючи раніше створений сервер у Azure. Після авторизації в акаунті Azure у Visual Studio проєкт було опубліковано в ресурсну групу SortGarbage шляхом функції Publish.
На порталі Azure після завершення всіх необхідних налаштувань API було успішно розгорнуто.

5.2.2 Перевірити та налаштувати роботу розгорнутої програмної системи
Під час тестування було перевірено доступність API та роботу функціональних ендпоінтів, підключення до бази даних, наявність всіх таблиць та їх структура, робота IoT, генерація даних та їх надсилання на сервер. Було також перевірено функції авторизації та адміністрування, де користувачі з різними ролями мають доступ до різної функціональності. 

5.3	Висновок
Під час виконання лабораторної роботи було розгорнуто розроблену програмну систему для автоматизації сортування сміття.
